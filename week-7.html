Week #7: Last Coding Sprint and Polishes

Steven Jethro Vitangcul

Sat 26 July 2025 • 1 min read

For our last week, we finalized our codes and the robot tests and continued making the documentation. I realized that the implementation for CPU and I/O usage monitoring was wrong as it didn’t work for multiple processes, so I had to modify it. I made a wrapper for the static methods in popup_checker.py since they are not normally loaded in robot tests. The warning from using the OCR was also suppressed to make the robot tests’ output cleaner. I also refactored and separated robot tests into multiple keywords so that it’s modular and looks better. Then, I tested the merged files and fixed the “File not found error”, which was caused by a skipped file change. During the presentation, the CPU and I/O usage was not working, so I had to fix it after. I got to make it work so then I added saving the file for auto input to trigger the I/O usage monitor. After that, I added one more checker to UsageMonitor.robot to see if there was CPU and I/O usage. Based on the documentation made by my classmate, I made README.md, ensuring it was concise and informative. Same goes for the PowerPoint presentation. I had fun and learned a lot during my internship in Open iT. We were also lucky to have their benefits provided to us. I enjoyed the seven weeks with them.
